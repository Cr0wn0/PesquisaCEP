Outros tópicos

  1. Hoisting é um comportamento do JavaScript onde variáveis e funções são movidas para o topo do seu contexto de execução antes da execução do código. No entanto, apenas as declarações são içadas, não as inicializações.
      2. console.log(x); // undefined var x = 5; console.log(x); // 5
  2. Arrow functions são uma forma mais concisa de escrever funções em JavaScript. Elas não possuem seu próprio this, o que as torna ideais para funções de callback.
      3. const soma = (a, b) => a + b; console.log(soma(2, 3)); // 5
      function normal() { this.valor = 1; setTimeout(function() { this.valor++; console.log(this.valor); // undefined }, 1000); } new normal();
      function arrow() { this.valor = 1; setTimeout(() => { this.valor++; console.log(this.valor); // 2 }, 1000); } new arrow();
  3. Desestruturação é uma expressão que permite desempacotar valores de arrays ou propriedades de objetos em variáveis distintas.
      4. const pessoa = { nome: "Alice", idade: 30 }; const { nome, idade } = pessoa; console.log(nome); // Alice console.log(idade); // 30
      const numeros = [1, 2, 3]; const [primeiro, segundo] = numeros; console.log(primeiro); // 1 console.log(segundo); // 2
  4. Closures são funções que têm acesso ao escopo de sua função pai, mesmo após a função pai ter sido executada. Elas permitem preservar o estado.
      5. function criarContador() { let contador = 0; return function() { contador++; return contador; }; }
      const contador1 = criarContador(); console.log(contador1()); // 1 console.log(contador1()); // 2 const contador2 = criarContador(); console.log(contador2()); // 1
