Respostas para as pesquisas

  1. Assincronismo é uma abordagem de programação que permite que o código continue sua execução sem esperar pela conclusão de operações demoradas, como chamadas a APIs ou leitura de arquivos. Isso é crucial para manter a responsividade das aplicações, especialmente na web.
  2. AJAX é uma técnica que permite que páginas da web carreguem dados de forma assíncrona, sem recarregar a página. Ele usa o objeto XMLHttpRequest.
      Exemplo: const xhr = new XMLHttpRequest(); xhr.open("GET", "https://api.exemplo.com/dados", true); xhr.onload = function () { if (xhr.status >= 200 && xhr.status < 300) { console.log(JSON.parse(xhr.responseText)); } }; xhr.send();
  3. Promises são objetos que representam o resultado de uma operação assíncrona, permitindo que você trate o resultado ou o erro quando estiver disponível.
      Exemplo: const minhaPromise = new Promise((resolve, reject) => { const sucesso = true; // Simula uma operação if (sucesso) { resolve("Operação bem-sucedida!"); } else { reject("Erro na operação."); } }); minhaPromise .then(result => console.log(result)) .catch(error => console.error(error));
  4. A Fetch API é uma interface moderna para fazer requisições assíncronas de rede, substituindo XMLHttpRequest. Ela retorna uma Promise que resolve para a resposta da requisição.
      Exemplo:fetch("https://api.exemplo.com/dados") .then(response => response.json()) .then(data => console.log(data)) .catch(error => console.error("Erro:", error));
  5. Async/Await é uma sintaxe que simplifica o trabalho com Promises, permitindo que o código assíncrono seja escrito de forma mais parecida com o código síncrono.
      const obterDados = async () => { try { const response = await fetch("https://api.exemplo.com/dados"); const data = await response.json(); console.log(data); } catch (error) { console.error("Erro:", error); } }; obterDados();
  • Desvantagens e vantagens:
  • AJAX:
      Vantagens: Funciona em todos os navegadores, fácil de implementar.
      Desvantagens: Sintaxe mais verbosa, difícil de gerenciar com várias requisições.
  • Promises:
      Vantagens: Facilita o tratamento de operações assíncronas, encadeamento fácil.
      Desvantagens: Pode levar a "callback hell" se não for gerenciado corretamente.
  • Fetch API:
      Vantagens: Sintaxe simples e baseada em Promises, suporte a operações mais modernas.
      Desvantagens: Não suporta abortar requisições (sem AbortController) e não é suportada em navegadores muito antigos.
  • Async/Await:
      Vantagens: Código mais limpo e legível, fácil de tratar erros com try/catch.
      Desvantagens: Requer um entendimento sólido de Promises, pode ser mais difícil de depurar.
